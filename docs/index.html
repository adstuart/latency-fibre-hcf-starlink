The model, in its entirety.

Everything on this page comes from the same handful of equations. They run client-side on every slider change, which is why the page is instantly interactive with no backend. 

// constants
const c      = 299_792.458;                    // km/s, vacuum
const v_smf  = c / n;                          // n = 1.468 for ITU-T G.652 silica[1]
const v_hcf  = 0.997 * c;                      // Microsoft / Lumenisity NANF[2]

// fibre one-way
const oneWay = cable_km / v + n_hops * 0.05;   // ~50 µs per router hop

// Starlink idealised (LEO + inter-satellite laser)
const arc    = gc_km * (R_earth + h) / R_earth;
const oneWay = (arc + 2*h) / c + n_isl * 1.0 + 2 * 2.0;

// Starlink realistic (ISL gap → terrestrial backhaul)
const oneWay = sat_access/c + backhaul_km/v_smf + 6.4;

Caveats.

  • Hollow-core isn't deployed on these routes yet. Microsoft have announced intra-datacentre and metro deployments[2]; trans-oceanic HCF is a "what if" scenario for now.
  • Starlink's ISL mesh is incomplete. Coverage is best within single hemispheres and across the North Atlantic; full London↔Sydney routing via pure ISL is still aspirational[3].
  • This is propagation, not the full stack. TCP handshake, TLS, congestion control and application parsing add real-world latency that dominates for short transfers.
  • Submarine cable routes vary. We use a representative 2025 production path length per corridor[4]; specific cables differ by ±15%.
  • Overhead is a budget, not a measurement. 50 µs per router hop × 8 hops is a reasonable long-haul placeholder; reality has jitter, queuing and reroutes.

Reproduce this.

git clone https://github.com/adstuart/latency-fibre-hcf-starlink
cd latency-fibre-hcf-starlink
python3 -m venv .venv && .venv/bin/pip install numpy

  2. Microsoft Azure blog, "Advancing networking with hollow-core fiber" (2023). Microsoft acquired Lumenisity in Dec 2022; NANF fibre operates at ~0.997 c with ~47% latency reduction vs silica.
  3. SpaceX Starlink Gen2 FCC filings (IBFS); public ISL RTT measurements on trans-Atlantic paths reported ~95 ms (Oct 2024). Pacific-rim ISL coverage continues to expand through 2025–2026.
  4. TeleGeography Submarine Cable Map. Representative London↔NYC: Grace Hopper 7,530 km, AEConnect 5,536 km, EXA Express 5,218 km. No direct London↔Sydney cable exists.
